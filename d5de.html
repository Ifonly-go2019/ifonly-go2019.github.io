Ezpop(Redo)

考点: php反序列化pop chain 构造…. em 可以说还有php伪协议的考点

我之前读不出来, 一方面是链子没打好,另一方面就是没用伪协议

各个魔法函数的作用我已经在源码中注释好了, 关键就是如何利用了, 即构造POP chian. 从以前的博客搬过来一句话,来回顾一下pop 链

把魔术方法作为最开始的小组件，然后在魔术方法中调用其他函数(小组件)，通过寻找相同名字的函数，再与类中的敏感函数和属性相关联，就是POP CHAIN 。此时类中所有的敏感属性都属于可控的。当unserialize()传入的参数可控，便可以通过反序列化漏洞控制POP CHAIN达到利用特定漏洞的效果。

提炼出关键点: 魔法方法, 其他函数, 相同名字的函数, 函数和属性相关联

构造pop 链:

  • 在类Show 中, $this->souce 如果是Show 类,就会调用__toString方法
  • __toString方法中 $this->str->source , 如果str为Test类中的属性p, 然后Test类中不存在source属性,就会调用__get方法
  • __get方法将属性p作为函数返回,如果属性p 是Modifier类就会调用__invoke方法, __invoke方法中就会去包含var,从而通过伪协议读取到flag.php

在var\Typecho\Db\Query.php 中的 532-563行中:

这里如果action为 SELECT 的话, __toString魔法函数就返回$this->_adapter->parseSelect($this->_sqlPreBuild); 在这个魔法函数里面,触发了_adapter的parseSelect方法, 如果我们让这里的_adapter实例化为SoapClitent对象, 然后调用parseSelect方法, 因为在这个类中这个方法不存在,所以就触发了__call方法, 因为这个__call方法里面的内容我们可以自己构造, 而Soap类也是PHP原生类中用于反序列化最好用的了

POP 链逻辑:

  1. 反序列化HelloWorld_DB 类, 触发__wakeup 方法, 在__wakeup方法中实例化了Typecho_Db对象
  2. PHP 的数组是可以储存对象的, 假设实例化Typecho_Db对象时 $this->coincidence['hello'] 实例化为Typecho_Db_Query 对象,在Typecho_Db构造方法中,将其作为字符串进行调用, 就会触发Typecho_Db_Query类中的__toString魔法方法.
  3. __toString 魔法方法中, 如果$_sqlPreBuild['aciton']为select 就会触发属性$_adapter的parseSelect()方法
  4. 将$_adapter实例化为SoapClient, 调用parseSelect是本类中不存在的方法, 故触发了__call魔术方法

exp: 几乎是照着Y1ng 师傅的exp打的…

直接生成的payload不会触发成功，要将字符串改写成十六进制，也就是将表示字符串的s写成大写S，这样private属性后面的%00这个不可见字符就能写成\00（如果是小写s 这个\00表示一个斜线和两个0 是三个字符）

www.gem-love.com

原理:

在 PHP5 最新的 CVS 中，
新的序列化方式叫做 escaped binary string 方式，这是相对与普通那种 non-escaped binary string 方式来说的：
string 型数据（字符串）新的序列化格式为：
S:"<length>":"<value>";
其中 是源字符串的长度，而非 <value> 的长度。<length> 是非负整数，数字前可以带有正号（+）。<value> 为经过转义之后的字符串。
它的转义编码很简单，对于 ASCII 码小于 128 的字符（但不包括 \），按照单个字节写入（与 s 标识的相同），对于 128~255 的字符和 \ 字符，则将其 ASCII 码值转化为 16 进制编码的字符串，以 \ 作为开头，后面两个字节分别是这个字符的 16 进制编码，顺序按照由高位到低位排列，也就是第 8-5 位所对应的16进制数字字符（abcdef 这几个字母是小写）作为第一个字节，第 4-1 位作为第二个字节。依次编码下来，得到的就是 <value> 的内容了
