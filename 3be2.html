[V&N2020 公开赛]TimeTravel

  • HTTPoxy漏洞（CVE-2016-5385）https://github.com/vulhub/vulhub/tree/master/cgi/httpoxy
  • 漏洞前提,服务器是在CGI模式下运行

简单来说，根据RFC 3875规定，cgi（fastcgi）要将用户传入的所有HTTP头都加上HTTP_前缀放入环境变量中，而恰好大多数类库约定俗成会提取环境变量中的HTTP_PROXY值作为HTTP代理地址。于是，恶意用户通过提交Proxy: http://evil.com这样的HTTP头，将使用缺陷类库的网站的代理设置为http://evil.com，进而窃取数据包中可能存在的敏感信息。

PHP5.6.24版本修复了该漏洞，不会再将Proxy放入环境变量中。本环境使用PHP 5.6.23为例。

当然，该漏洞不止影响PHP，所有以CGI或Fastcgi运行的程序理论上都受到影响。

https://github.com/vulhub/vulhub/tree/master/cgi/httpoxy

通过/phpinfo 得到php的版本为: 5.6.23

该漏洞在 PHP 5.6.24 之后修复了 不会将Proxy放入环境变量中

无列名注入

select 1,2,3; 时会创建一个虚拟表,这个表的列名为1,2,3

使用union向这个虚拟表中插入数据 select 1,2,3 union select * from users; 相当于将users 表改了个列名, 然后再使用select 将列选出来

select 2 from(select 1,2,3 union select * from users)a;

先生成一个列明为123的user表,当作a;然后选择2 这一列,得到数据

用反单引号代表列明(应该也可以不用)

利用 innodb 存储引擎

在MySQL 5.5.x 之后, 默认使用 innodb 存储引擎,5.6.x之后, innodb 添加了两个新表,innodb_index_stats 和 innodb_table_stats, 两个表都会存储数据库和对应的数据表,与information_schema 唯一不同的地方是没有列明,故需要结合无列名注入
